# Changelog

Все значимые изменения в проекте фиксируются в этом файле.

## [0.5.5.0] - 2026-04-07

### Added

- Интеграция с **облачным** провайдером (OpenAI-совместимый API `POST .../v1/chat/completions`): константы URL, ключа, модели, таймаута; общий модуль `ИАДО_ИнтеграцияOpenAIСовместимыйВызовСервера`.
- Перечисление провайдеров ИИ: **локальный Ollama** / **облако**; выбор в карточке эксперта.
- Обработка **«Настройки интеграции ИИ»**, общая команда открытия, подсистема **«Интеграция ИИ»**; тесты подключения Ollama и облака.
- Константа `ИАДО_ОблакоИИPDFТолькоТекст` (сохранена для совместимости; флаг в форме скрыт, логика анализа для облака не зависит от режима «файлом»).

### Changed

- Проект EDT расширения: **`DOCFSKDEVII2.ИИАнализДокументовOllama`** (префикс заимствований `ИАДО_` без изменений).
- **PDF в облаке**: текст готовится в базе (извлечение / распознавание), в API уходит **только текст** промпта (DeepSeek и аналоги не принимают `type: file` в `messages`).
- Определение расширения файла для справочника **Файлы**: используется `ТекущаяВерсияРасширение`, а не несуществующий реквизит `Расширение`.

### Fixed

- Форма настроек: чтение реквизита `РедактированиеДоступно` на клиенте (команды теста без ошибки «поле недоступно для чтения»).

## [0.4.29.0] - 2026-04-01

### Added

- Общий модуль `ИАДО_СценарийСхемыОбработкиОбъектаВызовСервера` — точка входа для выражения скрипта схемы обработки объекта (в т.ч. при расширении `lvv_ОтклБезопасныйРежимВыполненияСкриптов`).
- Заимствование `Справочник.ОбработкиОбъектов` с экспортной процедурой в модуле объекта (запасной вход для схемы при совпадении UUID с ИБ).
- Дополнение заимствования `Справочник.ДокументыПредприятия`: `ManagerModule.bsl` с дублёром вызова анализа (запасной вход).
- Заимствование общего модуля `РаботаСКомплекснымиБизнесПроцессамиСерверПереопределяемый` с прокси к запуску анализа и комментариями для схемы.
- Документ [`docs/DO3_SKILLS_INDEX.md`](./docs/DO3_SKILLS_INDEX.md) — индекс навыков сценариев ДО 3.0.

### Changed

- `ИАДО_ИнтеграцияИИВызовСервера`: уточнены комментарии по вызову из алгоритма схемы (безопасный режим / `ВыполнитьМетодКонфигурации` / `lvv`).
- Подсистема `ИАДО_ИнтеграцияИИ`, состав конфигурации расширения.
- Версия расширения `0.4.29.0`.

## [0.4.23.0] - 2026-04-01

### Added
- Регистр очереди распознавания `ИАДО_ОчередьРаспознаванияФайлов` (последовательный запуск распознавания при нескольких запросах).
- Дополнительное журналирование этапов обращения к модели (в т.ч. «начало» запроса).
- Документация: совместимость с ДО 3.0.19.30, [`docs/OLLAMA_SERVER.md`](./docs/OLLAMA_SERVER.md), [`docs/GIT_FOR_TEAM.md`](./docs/GIT_FOR_TEAM.md), ссылка на GitHub Releases в `README.md`.
- Общая картинка `ИАДО_ИнтеграцииИИOllama` (при выгрузке из конфигуратора в состав расширения).

### Changed
- Удалена интеграция **OpenRouter** (модуль, константы, команда теста); вызов модели только через **Ollama**.
- Перечисление `ИАДО_ПровайдерыИИ`: оставлено значение «Локальный Ollama».
- Запись журнала и очереди: режим дозаписи регистра (`Записать(Ложь)`) для сохранения истории.
- Версия расширения `0.4.23.0`.

### Fixed
- Выбор «головы» очереди распознавания по **актуальному** статусу заявки (последняя запись по идентификатору), без зависания из-за старых строк «Ожидает».

## [0.4.21.0] - 2026-03-31

### Added
- Добавлен модуль `ИАДО_ОтветИИФайлДокументаВызовСервера`.
- Реализовано создание DOCX-вложения с ответом модели и прикрепление к документу.
- Добавлено автосохранение результата анализа в карточку документа.

### Changed
- В `ИАДО_ИнтеграцияИИВызовСервера` добавлен вызов создания DOCX после успешного ответа модели.
- В `ИАДО_ЖурналИИВызовСервера` улучшена запись временной метки (`ТекущаяУниверсальнаяДата`) для стабильного журнала.
- Обновлена версия расширения до `0.4.21.0`.

### Fixed
- Исправлен сценарий подготовки текста: при отсутствии текста запускается штатное распознавание ДО.
- Реализовано ожидание появления текста версии файла с таймаутом и повторными попытками.
- Устранена серверная проблема с паузой опроса (`ОбщегоНазначенияБТС.Пауза` вместо недоступной глобальной `Приостановить`).

## [0.4.20.0] - 2026-03-31

### Added
- Фоновая операция анализа файла по кнопке из карточки документа.
- Интеграция с распознаванием ДО перед анализом ИИ.

### Changed
- Переработана логика подготовки текста для анализа с fallback на распознавание.
- Обновлена версия расширения до `0.4.20.0`.

## [0.4.1.0] - 2026-03-30

### Added
- Форма записи для правил экспертов с редактированием промпта.
- Кнопка ИИ-анализа в форме элемента `ДокументыПредприятия`.
- Базовый цикл интеграции с Ollama.
