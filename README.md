# ИИ-анализ документов (Ollama) для 1С:Документооборот

Расширение `DOCFSKDEVII.ИИАнализДокументовOllama` добавляет в 1С:Документооборот сценарий анализа файлов документа через локальный/сетевой Ollama и OpenRouter.

## Что реализовано

- Кнопка анализа в карточке `Справочник.ДокументыПредприятия`.
- Подбор эксперта ИИ по правилам (вид документа + тематика + приоритет).
- Вызов модели через:
  - Ollama (`/api/generate`),
  - OpenRouter (chat API).
- Журналирование обращений в `РегистрСведений.ИАДО_ЖурналОбращенийКИИ`.
- Автозапуск штатного распознавания ДО, если текст версии файла еще не готов.
- Ожидание появления текста с повторными попытками (polling + timeout).
- Сохранение ответа модели как DOCX-файла в карточку того же документа.

## Архитектура

Основные объекты расширения:

- `ИАДО_ИнтеграцияИИКлиент` — запуск операций из UI.
- `ИАДО_ИнтеграцияИИВызовСервера` — фоновый сценарий анализа файла.
- `ИАДО_ТекстФайлаДляИИВызовСервера` — подготовка текста из текущей версии файла.
- `ИАДО_ИнтеграцияOllamaВызовСервера` / `ИАДО_ИнтеграцияOpenRouterВызовСервера` — HTTP-клиенты провайдеров.
- `ИАДО_НастройкиАнализаИИВызовСервера` — настройки, подбор эксперта, тест подключения.
- `ИАДО_ЖурналИИВызовСервера` — запись событий в журнал обращений.
- `ИАДО_ОтветИИФайлДокументаВызовСервера` — сборка DOCX и прикрепление к документу.

## Быстрый старт

1. Подключите расширение `DOCFSKDEVII.ИИАнализДокументовOllama` к ИБ.
2. Заполните константы провайдера (`ИАДО_OllamaБазовыйURL`, таймаут и т.д.).
3. Создайте/активируйте экспертов в `Справочник.ИАДО_ЭкспертыИИ`.
4. Настройте правила в `РегистрСведений.ИАДО_ПравилаЭкспертовДокументов`.
5. В карточке документа выберите файл и нажмите кнопку ИИ-анализа.

## Ограничения и важные замечания

- Расширение не изменяет типовую конфигурацию напрямую — только механизм расширений.
- Для успешного анализа нужен текст текущей версии файла (извлечение текста/распознавание в ДО).
- `localhost` в URL Ollama означает сервер 1С, а не клиентский ПК.
