# ИИ-анализ документов (Ollama) для 1С:Документооборот

Расширение **«ИИ анализ документов (Ollama)»** (в EDT: проект `DOCFSKDEVII2.ИИАнализДокументовOllama`, в конфигурации — объекты с префиксом `ИАДО_`) добавляет в **1С:Документооборот** анализ файлов карточки документа через **локальный Ollama** и/или **облачный API** (OpenAI-совместимый `chat/completions`, напр. DeepSeek).

## Совместимость

- Расширение **разрабатывалось и тестировалось** на конфигурации **1С:Документооборот КОРП, редакция 3.0**, релиз платформы/конфигурации **3.0.19.30** (апрель 2026).
- На других ревизиях 3.0 и иных редакциях поведение может отличаться (имена элементов форм, подсистемы ДО) — требуется проверка.

## Сборки (релизы)

Готовый файл расширения **`.cfe`** для установки без сборки из исходников выкладывается в **GitHub Releases**:

**[Скачать релизы →](https://github.com/konstmax1388/AI-Document-Analysis--Ollama-/releases)**

Рекомендуется брать **последний** релиз по дате/версии и сверять номер с полем `version` в метаданных расширения.

## Ollama на сервере или ВМ

Запросы к модели выполняются **с сервера 1С**. Ollama нужно развернуть на хосте, доступном этому серверу по сети.

**[Подробная инструкция: развёртывание Ollama](./docs/OLLAMA_SERVER.md)**

Там же: замечание о недоступности исходной внешней ссылки (DeepSeek), сеть, файрвол, проверка `curl`, безопасность.

## Что реализовано

- Интеграция с карточкой `Справочник.ДокументыПредприятия` (команда и кнопки анализа создаются из модуля формы — см. инструкции).
- Подбор эксперта ИИ по правилам (вид документа + тематика + порядок).
- Вызов модели через **Ollama** (`/api/generate`) или **облако** (`/v1/chat/completions`).
- Журнал обращений: `РегистрСведений.ИАДО_ЖурналОбращенийКИИ`.
- При необходимости — штатное распознавание ДО и очередь распознавания перед анализом.
- Сохранение ответа модели как **DOCX** во вложения документа (в актуальных версиях расширения).

## Архитектура (кратко)

- `ИАДО_ИнтеграцияИИКлиент` — UI и длительные операции.
- `ИАДО_ИнтеграцияИИВызовСервера` — фоновый сценарий анализа файла.
- `ИАДО_ИнтеграцияOllamaВызовСервера` — HTTP-клиент Ollama.
- `ИАДО_НастройкиАнализаИИВызовСервера` — константы, эксперты, правила.
- `ИАДО_ЖурналИИВызовСервера`, `ИАДО_ОтветИИФайлДокументаВызовСервера`, `ИАДО_ТекстФайлаДляИИВызовСервера` — журнал, DOCX, текст файла.

## Быстрый старт

1. Установите расширение из [Releases](https://github.com/konstmax1388/AI-Document-Analysis--Ollama-/releases) или из файла в [`releases/`](./releases/) (см. [RELEASE_II_OLLAMA_2.0.md](./docs/RELEASE_II_OLLAMA_2.0.md)), либо соберите из `DOCFSKDEVII2.ИИАнализДокументовOllama`.
2. Разверните Ollama по [OLLAMA_SERVER.md](./docs/OLLAMA_SERVER.md) (если используете локальный провайдер), задайте константы / форму **«Настройки интеграции ИИ»**.
3. Настройте экспертов и регистр правил.
4. В карточке документа выполните анализ файла (см. [INSTRUCTIONS.md](./docs/INSTRUCTIONS.md)).

## Ограничения

- Расширение не меняет типовую конфигурацию напрямую — только механизм расширений.
- Для анализа нужен текст версии файла (извлечение / распознавание в ДО).
- `localhost` в URL Ollama указывает на **сервер 1С**, не на ПК пользователя.
