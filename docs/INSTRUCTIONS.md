# Инструкции по установке и использованию

## 1. Установка

### Требования

- 1С:Документооборот КОРП (рабочая конфигурация проекта).
- Подключенное расширение `DOCFSKDEVII.ИИАнализДокументовOllama`.
- Доступ сервера 1С к Ollama/OpenRouter по сети.

### Подключение расширения

1. Откройте конфигурацию в EDT/Конфигураторе.
2. Подключите расширение `DOCFSKDEVII.ИИАнализДокументовOllama`.
3. Обновите конфигурацию базы данных.
4. Проверьте, что в конфигурации видны объекты с префиксом `ИАДО_`.

## 2. Первичная настройка

### 2.1 Провайдер

Заполните константы:

- `ИАДО_OllamaБазовыйURL` (например `http://<host>:11434`),
- `ИАДО_OllamaТаймаутСек`,
- при использовании OpenRouter: `ИАДО_OpenRouterБазовыйURL`, `ИАДО_OpenRouterКлючAPI`.

### 2.2 Эксперты и правила

1. В `Справочник.ИАДО_ЭкспертыИИ` создайте экспертов.
2. Укажите модель, системный промпт, флаг использования.
3. В `РегистрСведений.ИАДО_ПравилаЭкспертовДокументов` задайте правила:
   - Вид документа,
   - Тематика,
   - Эксперт,
   - Порядок применения,
   - Промпт обработки документа.

## 3. Работа пользователя

1. Откройте карточку документа (`ДокументыПредприятия`).
2. Выберите файл документа.
3. Нажмите кнопку ИИ-анализа.
4. Дождитесь завершения фоновой операции.
5. Проверьте результат:
   - текст ответа в форме,
   - новый DOCX-файл во вложениях документа.

## 4. Что происходит в фоне

1. Подбирается эксперт по правилам.
2. Берется текст текущей версии файла.
3. Если текста нет — запускается штатное распознавание ДО.
4. Выполняются повторные попытки чтения текста до таймаута.
5. Отправляется запрос в модель.
6. Записывается журнал обращения в `ИАДО_ЖурналОбращенийКИИ`.
7. Формируется и прикрепляется DOCX с ответом.

## 5. Диагностика проблем

### Нет ответа модели

- Проверьте URL/таймаут и сетевую доступность Ollama.
- Проверьте активность эксперта и наличие правила под вид/тематику.
- Проверьте наличие текста в `РегистрСведений.ТекстыВерсийФайлов`.

### Ошибка открытия вложения

- Убедитесь, что файл создан после актуального релиза (`>=0.4.21.0`).
- Удалите старые тестовые вложения, созданные до фикса.
- Повторите анализ и откройте новый DOCX.

### В журнале видна не вся история

- Сбросьте отборы формы списка регистра.
- Проверьте права/ограничения доступа пользователя.
- Сверьте количество строк запросом к регистру.

## 6. Эксплуатационные рекомендации

- Используйте отдельную модель под каждый тип документов (через экспертов).
- Не отправляйте в промпт лишние персональные данные.
- Для тяжелых сканов увеличивайте таймауты и контролируйте нагрузку OCR.
- Храните ключи API (OpenRouter) в защищенном контуре.
