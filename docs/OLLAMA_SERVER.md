# Развёртывание Ollama на сервере или виртуальной машине

Документ предназначен для администраторов: поднять **Ollama** в контуре, доступном **серверу 1С** (не рабочему месту пользователя). Исходная [ссылка на инструкцию DeepSeek](https://chat.deepseek.com/share/mxbcl23zkqujl4yj02) при проверке **недоступна** (ответ CDN 403), поэтому ниже приведена **актуализированная** схема по официальной документации Ollama и типовым практикам Linux.

Официальные материалы (проверяйте версию под вашу ОС):

- [https://github.com/ollama/ollama](https://github.com/ollama/ollama) — установка, релизы.
- [https://ollama.com/download](https://ollama.com/download) — быстрые команды установки.

## 1. Рекомендации перед установкой

| Тема | Рекомендация |
|------|----------------|
| Где ставить | Отдельная ВМ или сервер в той же сети, что и кластер 1С (или с маршрутом/файрволом до него). |
| URL в 1С | В константе `ИАДО_OllamaБазовыйURL` указывайте **хост:порт**, с которого **видит сервер 1С** (`http://ollama-srv:11434`), а не `localhost` клиента. |
| GPU | Для больших моделей — NVIDIA GPU и драйверы на хосте Ollama; иначе только CPU (медленнее). |
| Память | Заложите RAM с запасом под размер модели (см. страницы моделей на ollama.com). |
| Безопасность | Порт `11434` по умолчанию **без встроенной авторизации**. Ограничьте доступ файрволом/VLAN только для серверов 1С; не выставляйте в интернет без reverse-proxy и политики доступа. |

## 2. Linux (сервер / ВМ): установка через официальный скрипт

Типовой путь для актуальных дистрибутивов (см. актуальную строку на [ollama.com/download/linux](https://ollama.com/download)):

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

После установки сервис обычно поднимается автоматически. Проверка:

```bash
ollama --version
systemctl status ollama
```

Скачивание и запуск модели (пример — уточните имя под вашу задачу):

```bash
ollama pull llama3.2
ollama run llama3.2 "ping"
```

Проверка HTTP API с той же машины:

```bash
curl http://127.0.0.1:11434/api/tags
```

## 3. Сеть и файрвол

- Откройте **входящий** TCP **11434** (или иной порт, если меняли) **только** для IP серверов 1С.
- С сервера приложений 1С выполните:

```bash
curl -sS -o /dev/null -w "%{http_code}" http://<хост-Ollama>:11434/api/tags
```

Ожидается код **200**.

## 4. Windows Server / ВМ под Windows

- Установите Ollama с [ollama.com/download/windows](https://ollama.com/download/windows) (установщик).
- Служба слушает `11434` локально; для доступа с других машин настройте **брандмауэр Windows** (разрешить входящий TCP 11434 с подсети 1С) и при необходимости переменные окружения сервиса (см. документацию Ollama для привязки к `0.0.0.0`).

## 5. Docker (по желанию)

Если используете контейнер — образ и параметры см. в README проекта Ollama на GitHub. Важно:

- Проброс порта `-p 11434:11434`.
- Том для кэша моделей (volume), иначе образ пересоздаётся — модели качаются заново.
- Ограничение доступа сетью к хосту Docker.

## 6. Связка с 1С:Документооборот и расширением ИАДО

1. Убедитесь, что с **рабочего процесса сервера 1С** (rphost) доступен `http://<ollama>:11434`.
2. В константе `ИАДО_OllamaБазовыйURL` укажите тот же базовый URL (без завершающего `/`, как принято в коде расширения).
3. В подсистеме расширения выполните **«Тест подключения Ollama»** (общая команда) или настройте эксперта и правило и выполните анализ тестового файла.

## 7. Эксплуатация

- Обновление Ollama: следуйте инструкциям дистрибутива (`apt upgrade`, переустановка MSI и т.д.).
- Список моделей на сервере: `ollama list`; удаление неиспользуемых — освобождает диск.
- Мониторинг: нагрузка CPU/GPU, место на диске, задержки ответа API.

Если понадобится восстановить исходную идею из недоступной внешней инструкции, сохраните её текст локально в корпоративной wiki и приложите к этому файлу ссылку на **внутреннюю** копию.
