# ИИ-анализ документов (расширение ИИАнализДокументовOllama) — версия 0.4.1.0

## Границы работ (важно для доработок)

- **Трогаем только** расширение `DOCFSKDEVII.ИИАнализДокументовOllama` и при необходимости `DOCFSKDEVII.ДоработкиИнтерфейса`.
- **Основную конфигурацию** `DOCFSKDEVII` и **другие расширения** в репозитории **не изменяем** без отдельного решения заказчика.

## Кратко: что сделано

### 1. Промпт в правилах (`РегистрСведений.ИАДО_ПравилаЭкспертовДокументов`)

Текст промпта по-прежнему хранится в ресурсе **ПромптОбработкиДокументаХранилище**. Для ввода добавлена **форма записи регистра**:

- В метаданных регистра указаны `defaultRecordForm` и форма **ФормаЗаписи**.
- Файлы: `InformationRegisters\ИАДО_ПравилаЭкспертовДокументов\Forms\ФормаЗаписи\Form.form` и `Module.bsl`.
- На форме: измерения (вид, тематика), эксперт, порядок и многострочное поле **«Промпт обработки документа»** (реквизит `ИАДО_ТекстПромпта`). При открытии значение читается из хранилища, при записи — записывается в `ПромптОбработкиДокументаХранилище`.
- Открытие: список регистра → запись — откроется эта форма; промпт редактируется явно.

### 2. Кнопка в карточке документа (расширение `DOCFSKDEVII.ИИАнализДокументовOllama`)

- В `Catalogs\ДокументыПредприятия\ДокументыПредприятия.mdo` добавлено заимствование **ФормаЭлемента** с **собственным** `uuid` формы (`7c3a91b2-…`), чтобы не пересекаться с **ДоработкиИнтерфейса** (`652d0c00-…`).
- Файл `Forms\ФормаЭлемента\Module.bsl`:
  - после `ПриСозданииНаСервере` создаётся команда и кнопка **«ИИ: проанализировать файл»** с картинкой `БиблиотекаКартинок.АнализДанных48`, отображение «картинка и текст», `ВажностьПриОтображении.ОченьВысокая`, вставка перед «Записать и закрыть»;
  - видимость — `ИАДО_НастройкиАнализаИИВызовСервера.ЕстьПравилоДляПередачиВИИПоВидуДокументу(Объект.ВидДокумента)` (только явный вид с активным правилом);
  - обновление видимости: после `ПослеЗаписиНаСервере`, `ПриЧтенииНаСервере`, `ПриИзмененииВидаДокумента`;
  - по кнопке: документ должен быть записан; в дереве **Файлы** — строка файла с заполненной **Ссылкой**; запуск `ИАДО_ИнтеграцияИИКлиент.ВыполнитьОперациюИИСОжиданием` с `АнализФайла`; по успеху — `ПоказатьЗначение` с текстом ответа модели.

### 3. Версия

- В `Configuration.mdo` расширения версия **0.4.1.0**.

## Как быстро прогнать сценарий

1. Обновить конфигурацию ИБ из EDT / загрузить расширение.
2. В **«Правила выбора эксперта…»** создать или открыть запись: заполнить вид (не «любой»), эксперта, при необходимости тематику — внизу отредактировать промпт, **записать**.
3. Открыть **Документы предприятия** с этим видом: на командной панели должна быть акцентная кнопка ИИ.
4. Выделить в списке **конкретный файл** (не группу), нажать кнопку — дождаться фона и окна с ответом.

### Важно про два расширения на одной форме

Если одновременно подключены и **ДоработкиИнтерфейса**, и **ИИАнализДокументовOllama**, обе заимствуют одну и ту же **ФормаЭлемента** документа — итог зависит от **порядка расширений** в конфигурации; при сбоях имеет смысл проверить порядок подключения.

## Проверка EDT

Ошибок уровня **ERRORS** нет; предупреждения по путям данных у заимствованной **ФормаЭлемента** — типичный шум EDT для расширений без полной базовой конфигурации в одном проекте; на рантайм обычно не влияет.

## Форма документа и XDTO (платформа 8.3.27)

При ошибке загрузки `Form.xml` по свойству **Picture**: в расширении должны быть корректно заполнены блоки **PictureDecorationExtInfo** (не пустые, с вложенной картинкой) и у команд с **TextPicture** / **PictureAndText** — элемент **picture** там, где его ожидает схема XDTO. Правки такого рода выполняются **только внутри** `DOCFSKDEVII.ИИАнализДокументовOllama` (файлы `Form.form` / `BaseForm\Form.form` заимствованной формы), без правок основной конфигурации и чужих расширений.
