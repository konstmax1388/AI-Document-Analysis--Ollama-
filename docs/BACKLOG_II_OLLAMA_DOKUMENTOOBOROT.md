# Бэклог: ИИ-анализ файлов документов через Ollama

## Цель

Сделать устойчивый и управляемый процесс ИИ-анализа файлов документов в 1С:Документооборот:

- запуск из карточки документа,
- выбор эксперта по бизнес-правилам,
- получение ответа модели,
- сохранение результата в документ (в том числе DOCX-вложением),
- аудит всех обращений.

## Планы (дорожная карта)

Задачи средне- и долгосрочной перспективы; приоритеты и состав уточняются с заказчиком.

### Облачные ИИ

- [ ] **Реализация поддержки облачных ИИ** — подключение внешних API (типа OpenAI-совместимых, прочих облачных моделей): модель безопасности (ключи, хранилище паролей БСП), выбор провайдера в эксперте/правиле, таймауты и лимиты, журналирование, согласование с политикой выноса данных за контур.

### Автоматизация процесса

- [ ] **Автоматическое направление файлов на анализ при обработке документа** — триггеры на этапах жизненного цикла документа ДО (после регистрации, смены состояния, появления версии файла и т.д.): правила «когда запускать», идемпотентность, очередь/фон, уведомления пользователю и обработка ошибок без блокировки основного процесса.

### Развитие продукта

- [ ] **Общее развитие продукта** — улучшение UX, сценарии для разных ролей, стабильность и производительность, документация и онбординг, метрики использования, обратная связь пользователей, подготовка к публичному репозиторию/релизам (по мере готовности).
