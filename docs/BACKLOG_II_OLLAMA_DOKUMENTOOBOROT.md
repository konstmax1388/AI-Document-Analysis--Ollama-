# Бэклог: ИИ-анализ файлов документов через Ollama

## Контекст

- **Конфигурация:** 1С:Документооборот КОРП, редакция 3.0, версия 3.0.19.30.
- **Правило доработок:** только расширения. UI и заимствования по **`ДокументыПредприятия`** — в расширении **`DOCFSKDEVII.ДоработкиИнтерфейса`**; прикладная логика ИИ, настройки, HTTP к Ollama — в отдельном расширении **`DOCFSKDEVII.ИИАнализДокументовOllama`** (префикс **`ИАДО_`**).
- **Ollama:** базовый URL в настройках (сейчас `http://192.168.10.79:11434`, позже ВМ Hyper-V).

**Сводка по текущему релизу расширения ИИ (что сделано, как проверить, границы правок):** [ИАДО_ИИ_0.4.1_что_сделано_и_проверка.md](./ИАДО_ИИ_0.4.1_что_сделано_и_проверка.md).

## Зафиксированные решения по данным

| Сущность | Объект метаданных | Реквизит / регистр |
|----------|-------------------|---------------------|
| Документ | `Справочник.ДокументыПредприятия` | — |
| Вид | `ВидДокумента` | `СправочникСсылка.ВидыДокументов` |
| Тематика | `Тематика` | `СправочникСсылка.ТематикиДокументов` |
| Файлы владельца | `Справочник.Файлы` | `ВладелецФайла` ∈ `ОпределяемыйТип.ВладелецФайлов` (вкл. `ДокументыПредприятия`) |
| Текст **текущей версии** | `РегистрСведений.ТекстыВерсийФайлов` | измерение `Версия` = `Файл.ТекущаяВерсия`; текст: `РегистрыСведений.ТекстыВерсийФайлов.ПолучитьТекстХ ...` / содержимое хранилища |
| Первый целевой вид (данные ИБ) | «Договор (нетип.)» | ссылка `ВидыДокументов` — получать в рантайме (`НайтиПоНаименованию` / код), не хардкодить UUID |

## Функциональные требования

1. **Настройки подключения:** базовый URL Ollama, таймаут (и при необходимости дорасширить — прокси, заголовки).
2. **Эксперты ИИ:** несколько записей с собственным промптом, именем, моделью, признаком активности.
3. **Соответствие вид + тематика → эксперт:** матрица в настройках с приоритетами (точное совпадение → только вид → эксперт по умолчанию — уточнить при реализации).
4. **Сценарий:** по документу собрать файлы с `ВладелецФайла = документ`, для каждого взять **текущую версию**, прочитать извлечённый текст, сформировать запрос к Ollama (`/api/generate` или `/api/chat`, `stream: false`).
5. **Результат:** ответ модели сохранить в **DOCX**, добавить как файл к **тому же** документу (типовые механизмы работы с файлами ДО).
6. **Безопасность:** учёт доступа к документу/файлам; не передавать лишнее в промпт; размер текста и таймауты.

## Этапы реализации и тестирования

### Этап 1 — каркас расширения и связь с ИБ (текущий)

- [x] Документ бэклог.
- [x] Проект `DOCFSKDEVII.ИИАнализДокументовOllama`: `Configuration.mdo`, `DT-INF/PROJECT.PMF`, `.project`.
- [x] Общий модуль `ИАДО_ИнтеграцияOllamaВызовСервера`: проверка доступности сервиса (HTTP `GET /api/tags`).
- **Тест 1.1 (ручной):** в EDT подключить расширение к основному проекту `DOCFSKDEVII`, обновить конфигурацию ИБ, включить расширение.
- **Тест 1.2:** на сервере 1С доступен Ollama; с пустыми константами — `ИАДО_ИнтеграцияOllamaВызовСервера.ПроверитьДоступность("http://192.168.10.79:11434", 30)`; после заполнения констант — `ИАДО_НастройкиАнализаИИВызовСервера.ПроверитьПодключениеПоКонстантам()`.

### Этап 2 — метаданные настроек (расширение ИИ)

- [x] Справочник **`ИАДО_ЭкспертыИИ`**: наименование/код (стандартные реквизиты), **`ИмяМоделиOllama`**, **`СистемныйПромптХранилище`**, **`Используется`**.
- [x] Регистр сведений **`ИАДО_ПравилаЭкспертовДокументов`**: измерения **`ВидДокумента`**, **`ТематикаДокумента`** (пустая ссылка = «любой»), ресурсы **`Эксперт`**, **`ПорядокПрименения`** (меньше — выше приоритет).
- [x] Константы **`ИАДО_OllamaБазовыйURL`**, **`ИАДО_OllamaТаймаутСек`**.
- [x] Подсистема **`ИАДО_ИнтеграцияИИ`** (без вывода в командный интерфейс).
- [x] Общий модуль **`ИАДО_НастройкиАнализаИИВызовСервера`**: `ПодобратьЭксперта`, `ПараметрыЭксперта`, `ПроверитьПодключениеПоКонстантам`, чтение констант.
- [ ] Форма рабочего места настроек (обработка) — по желанию; пока стандартные формы списка/элемента в конфигураторе/EDT.
- **Тест 2.1:** задать константу URL, создать 1–2 эксперта (промпт в хранилище через форму элемента), завести правила с разным `ПорядокПрименения`; в консоли: `ИАДО_НастройкиАнализаИИВызовСервера.ПодобратьЭксперта(Вид, Тематика)`, `ПараметрыЭксперта(ссылка)`, `ПроверитьПодключениеПоКонстантам()`.

### Этап 3 — сбор текста и вызов generate

- Запрос файлов по `ВладелецФайла`, обход `ТекущаяВерсия`, чтение `ТекстыВерсийФайлов`; обработка пустого текста и статуса извлечения.
- Сбор JSON тела под Ollama, разбор ответа, контроль ошибок HTTP.
- **Тест 3.1:** документ «Договор (нетип.)» с одним файлом, извлечённый текст не пустой — получить непустой ответ модели (на тестовой модели).

### Этап 4 — DOCX и прикрепление к документу

- Формирование DOCX (шаблон OOXML / `ФорматированныйДокумент` — выбрать после `codesearch` по типовым модулям).
- Вызов API ДО для создания файла с владельцем `ДокументыПредприятия`.
- **Тест 4.1:** в карточке документа виден новый файл, открывается в Word, текст совпадает с ответом.

### Этап 5 — UI в `ДоработкиИнтерфейса`

- Команда формы `ДокументыПредприятия`: «Анализ ИИ (Ollama)» / прогресс, вызов сервера расширения ИИ.
- **Тест 5.1:** полный цикл из интерфейса под пользователем с правами на документ и файлы.

## Риски и заметки

- `localhost` в URL настройки ссылается на **сервер 1С**, не на клиент; для сетевого Ollama указывать IP/имя ВМ.
- Режим совместимости расширения: **8.3.27** (как у основной конфигурации в выгрузке).
