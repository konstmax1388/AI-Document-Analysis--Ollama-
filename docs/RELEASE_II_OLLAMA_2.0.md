## Установка

1. Конфигуратор: **Конфигурация → Расширения конфигурации → Загрузить из файла** — указать `ИИАнализДокументовOllama2.0.cfe`.
2. Выполнить **Обновление конфигурации БД** для подключённого расширения.
3. Назначить пользователям роли расширения (`ИАДО_Просмотр`, `ИАДО_Использование`, при необходимости `ИАДО_ПолныеПрава`).

Подробнее: [`INSTRUCTIONS.md`](./INSTRUCTIONS.md).

## Ключевые возможности этой линии (относительно ранних 0.4.x)

- **Локальный Ollama** (`/api/generate`) — без изменения базовой идеи.
- **Облако** — вызов **OpenAI-совместимого** API **`/v1/chat/completions`** (в т.ч. DeepSeek): URL, ключ, модель, увеличенный таймаут.
- **Единая форма настроек** — обработка `ИАДО_НастройкиИнтеграцииИИ`, раздел интерфейса **«Интеграция ИИ»**.
- У эксперта — выбор **провайдера** (локальный Ollama / облако); правила по виду и тематике документа — как прежде.

## Облако и PDF

Провайдеры вроде **DeepSeek** принимают в `messages[].content` только **текстовые** части. **Передача PDF «файлом» (base64, type `file`) в chat API не используется.**  
Для PDF в облаке текст **подготавливается в базе** (извлечение / при необходимости штатное распознавание ДО), затем отправляется в модель **в промпте**, аналогично DOC/DOCX.
